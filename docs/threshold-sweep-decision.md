# 閾値スイープ（EDGE_MIN × ODDS_MAX）の方針メモ

作成日: 2026-07-12
関連スクリプト: [`scripts/sim-thresholds.ts`](../scripts/sim-thresholds.ts)（読み取り専用・本番非改変）
対象データ: v1 バックテスト（結果あり 216R / うちベースライン対象 214R）、`EV_MIN=0.10` 固定

## 結論

**現時点の有力候補: `EDGE_MIN=0.02` × `ODDS_MAX=20`**

| 指標 | ベースライン(0.02×50) | 候補(0.02×20) | 変化 |
|---|---:|---:|---:|
| ROI | −20.2% | **−7.4%** | **+12.8pt** |
| 対象レース数 | 214R | 131R | 61%残存（標本十分） |
| 的中率 | 12.1% | 9.9% | −2.2pt |
| 賭け点数 | 727 | 217 | 約70%削減 |

ODDS_MAX を 50→20 に下げるだけで ROI が大きく戻る＝**favorite-longshot bias（人気薄の過大評価）を踏み抜いていた**ことの裏付け。
ODDS_MAX=30 では改善ゼロ（人気薄は 20〜50 に集中）、15 まで絞ると逆に悪化（−33%）。**20 が明確なスイートスポット。**

## 見送った理由（EDGE_MIN≥0.05 系）

ROI が見かけ上大きく改善するケースがあったが、標本が極端に少なくノイズと判断して不採用。

- `EDGE_MIN=0.08 × ODDS_MAX=50`: ROI **+482.5%** だが **対象3R・点数4** ＝ 完全なノイズ。
- `EDGE_MIN=0.05 × ODDS_MAX=30`: ROI **+14.8%**（唯一プラス圏）だが **対象31R・点数50** ＝ 的中1〜2発で±40pt振れる規模。統計的に信頼不可。
- `EDGE_MIN=0.10`: どの ODDS_MAX でも **対象0R** ＝ 絞りすぎで機能停止。

**判断基準: 賭け点数が約50点（≈30R）を切ると ROI は信頼できない。**

## 今後の対応

1. **本番（`app/page.tsx` のフィルタ）はまだ変更しない。**
2. 来週末以降（2026-07-18 週〜）、`trainingScore` / `jockeyScore` の **v2 データが蓄積され次第**、同じ EDGE_MIN × ODDS_MAX スイープを **v2 データでも実行**し、同じ傾向（ODDS_MAX=20 付近が改善のスイートスポット）が再現するか確認する。
3. v2 データでも同様の傾向が確認できたら、本番の `ODDS_MAX` を **50→20 に変更**することを検討する。
4. その際、本スクリプト `scripts/sim-thresholds.ts` を再利用できる状態に保つ（下記「再実行手順」参照）。

## 再実行手順

```bash
npx tsx scripts/sim-thresholds.ts
```

- 集計バージョンは `TARGET_VERSION`（現在 `"v1"`）で切り替える。v2 で回すときは `"v2"` に変更。
- 閾値の組み合わせは `EDGE_MINS` / `ODDS_MAXS` 配列で調整可能。`EV_MIN` は据え置き前提。
- 出力は「対象R / 点数 / 的中率 / ROI / ROI改善 / 対象R残率」の一覧。ベースライン(0.02×50)との差分で読む。
- 標本を増やしたい場合は、Pass2（過去週データ）を含めるか v1+v2 合算で再サンプリングするとノイズ判定が固まる。

## 再検証（2026-07-13）: 馬番・オッズ紐付けバグ修正後

コミット `32c2d1c` で出馬表パースの馬番・オッズ紐付けバグを修正し、races-cache を是正
（主に v1 の 7/05 で 27 レース、6/28 で 1 レース）した。**上記の初回結論は「バグで
オッズが誤混入した汚染データ」に基づいていた**ため、是正後データ（v1・結果あり 216R）で
同一条件（`EV_MIN=0.10` 固定、v1、16通り）で再実行した。

### 結論：方向性は不変（むしろ強化）、ただし絶対 ROI は全面的に悪化

**バグは見かけの回収率を水増ししていた。** 是正後は全組み合わせで ROI が下がり、
「本命の少数に絞る（ODDS_MAX=20）」効果は**相対的に一層大きく**なった。

| 指標 | ベースライン(0.02×50) | 候補(0.02×20) | 変化 |
|---|---:|---:|---:|
| ROI | **−47.3%**（旧 −20.2%） | **−18.2%**（旧 −7.4%） | **+29.1pt**（旧 +12.8pt） |
| 対象レース数 | 213R（旧 214R） | 136R（旧 131R） | **64%残存**（標本十分） |
| 的中率 | 8.9%（旧 12.1%） | 8.8%（旧 9.9%） | −0.1pt |
| 賭け点数 | 729 | 225 | 約69%削減 |

- **ODDS_MAX=20 が引き続き明確なスイートスポット**。改善幅は +12.8pt → **+29.1pt** に拡大し、
  標本も 64%（136R）残る最大標本の改善。favorite-longshot bias の裏付けは一層強固。
- **ただし絶対 ROI は大幅悪化**：ベースライン −20.2%→**−47.3%**、候補 −7.4%→**−18.2%**。
  是正後データはエッジが想定より弱く、バックテスト上はより深いマイナス圏。

### 旧「見送り候補」の是正後の姿（＝バグ由来だったことの確認）

| 組み合わせ | 旧 ROI（対象R） | 新 ROI（対象R） | 判定 |
|---|---:|---:|---|
| 0.05×30 | **+14.8%**（31R） | **−33.1%**（39R） | 唯一プラス圏だった案が崩壊 → **バグ由来のノイズ確定** |
| 0.08×50 | **+482.5%**（3R） | **+4.4%**（7R） | 異常値は完全消滅 → **純粋なバグ由来アーティファクト** |
| 0.08×20〜15 | −100%（1R） | +34.3〜+88.0%（4〜5R） | 依然 標本 4〜7 点＝ノイズ、不採用 |
| 0.10×全て | 0R | 1〜2R・ROI −100% | 変わらず絞りすぎ |

- 初回に **0.05×30 や 0.08 系を採用しなかった判断は正しかった**（すべてバグ由来の見かけ改善）。
- 「賭け点数 ≈50点（≈30R）未満は ROI 信頼不可」の判断基準は是正後も有効。

### 今後の対応（更新）

1. **本番（`app/page.tsx` のフィルタ）はまだ変更しない**（据え置き）。
2. 有力候補は引き続き **`EDGE_MIN=0.02` × `ODDS_MAX=20`**。ただし是正後の絶対 ROI が
   −18.2% と深いため、**「回収率改善」ではなく「損失・分散の圧縮＋人気薄バイアス回避」**として
   位置づけを更新（プラス転換には別施策が要る）。
2026-07-18 週〜の **v2 データでの再スイープ**は予定どおり実施し、同傾向（ODDS_MAX=20）を確認する。
