## 実施内容

### 1. コアエンジン実装 (`lib/engine.ts`)

- **Bradley-Terry Latent Strength Model** を実装
- パイプライン：features → latent strength → BT確率 → 市場比較 → edge → EV → ランキング
- latent strength = `0.35×form + 0.25×training + 0.20×pedigree + 0.20×jockey`
- Bradley-Terry確率 = `exp(strength_i) / Σexp(strength_j)`（softmax）
- 市場確率はvig除去のため正規化（`(1/odds) / Σ(1/odds)`）
- EV = `modelProb × odds − 1`
- edge = `modelProb − marketProb`
- `valueRating` = `edge × 100 + ev × 10` によるランキングスコア

### 2. バックテスト実装 (`lib/backtest.ts`)

- `runBacktest(races)` 関数を実装
- 出力：`totalRaces`, `roi`, `totalReturn`, `avgEV`, `avgEdge`, `hitRateTop1`, `hitRateTop3`

### 6. UI 実装 (`app/page.tsx`)

`ui_design.png` を参考に全面実装。3行レイアウト。

#### Row 1
| パネル | 内容 |
|---|---|
| Panel A | VALUE HORSE ロゴ・週間成績（勝率/回収率/的中率）・SVGスパークライン |
| Panel B | レース選択ドロップダウン・SVG円形ゲージ（推奨馬の勝率）・レース詳細 |
| Panel C | EVランキングリスト・EVバー・クリックで馬選択 |

#### Row 2
| パネル | 内容 |
|---|---|
| Panel D | 選択馬名・EV値・SVGレーダーチャート（フォーム/血統/調教/騎手/乖離）・4指標ボックス |
| Panel E | AI分析テキスト（EV値に応じて◎/△/✕）・信頼度バー・EVランク |
| Panel F | 全馬スコア表（勝率/Edge/EV、EV降順） |

#### Row 3（参照買い目）
| 買い目 | ロジック |
|---|---|
| 単勝 | EV > 0 の馬をEV降順。Kelly比率 = `EV / (odds − 1)` |
| ワイド | モデル確率上位5頭の2頭ペアをpTrue合算降順 |
| 馬連 | EV上位5頭の2頭ペアをEV合算降順 |
| 三連複 | EV1位軸固定、相手をEV上位から総当たり |
| 三連単 | EV1位を1着固定、2〜3着をEV上位から順列 |

マーク（◎○▲△×）はランク順で付与。レース切り替え時に自動再計算。
